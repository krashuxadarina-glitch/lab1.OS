1. Словник базових англійських термінів
Virtual Machine (VM) — віртуальна машина; програмна імітація фізичного комп’ютера з власною операційною системою.
Virtualization — віртуалізація; технологія створення та використання віртуальних обчислювальних ресурсів.
Hypervisor — гіпервізор; програмне забезпечення, яке забезпечує створення, керування та виконання віртуальних машин.
Type 1 Hypervisor (Bare-metal) — гіпервізор першого типу; працює безпосередньо на апаратному забезпеченні без хостової ОС.
Type 2 Hypervisor (Hosted) — гіпервізор другого типу; працює поверх встановленої операційної системи.
Host Operating System — хостова операційна система, на якій запущений гіпервізор.
Guest Operating System — гостьова операційна система, що працює у віртуальній машині.
Virtual Disk — віртуальний диск; файл, який імітує фізичний накопичувач.
2. Поняття «гіпервізор» та його типи
Гіпервізор — це спеціалізоване програмне забезпечення, призначене для створення, запуску та керування віртуальними машинами. Він забезпечує розподіл апаратних ресурсів між кількома гостьовими операційними системами та їх взаємну ізоляцію, що дозволяє одночасно використовувати декілька ОС на одному фізичному комп’ютері.
Розрізняють два основні типи гіпервізорів:
Гіпервізор першого типу (Type 1, bare-metal) працює безпосередньо на апаратному забезпеченні комп’ютера та не потребує хостової операційної системи. Характеризується високою продуктивністю та надійністю і використовується переважно в серверних середовищах.
Гіпервізор другого типу (Type 2, hosted) працює як прикладна програма поверх хостової операційної системи. Є зручним для навчання, тестування та роботи з кількома ОС на персональних комп’ютерах.
2.2. Основні компоненти та можливості гіпервізора Hyper-V
(варіант 15)
Hyper-V — це гіпервізор першого типу (Type 1), розроблений компанією Microsoft, який використовується для створення та керування віртуальними машинами на платформах Windows.
Основні компоненти гіпервізора Hyper-V
Hyper-V Hypervisor — базовий рівень віртуалізації, який працює безпосередньо з апаратним забезпеченням і забезпечує ізоляцію віртуальних машин.


Parent Partition (батьківський розділ) — спеціальне середовище з ОС Windows, яке відповідає за керування віртуальними машинами та взаємодію з апаратними ресурсами.


Child Partitions (дочірні розділи) — середовища, у яких працюють гостьові операційні системи.


Virtual Machine Management Service (VMMS) — служба керування віртуальними машинами (створення, запуск, зупинка, налаштування).


Virtual Switch — віртуальний комутатор для організації мережевої взаємодії між віртуальними машинами та фізичною мережею.


Integration Services — набір драйверів і служб для покращення взаємодії гостьової ОС з гіпервізором.


Virtual Storage (VHD/VHDX) — система віртуальних жорстких дисків для зберігання даних віртуальних машин.
Основні можливості гіпервізора Hyper-V
створення та керування кількома віртуальними машинами;


підтримка різних гостьових операційних систем (Windows, Linux);


ефективний розподіл процесорних, пам’яттєвих і дискових ресурсів;


ізоляція віртуальних машин для підвищення безпеки;


підтримка знімків стану віртуальних машин (checkpoints);


віртуалізація мережі за допомогою віртуальних комутаторів;


висока продуктивність завдяки роботі безпосередньо з апаратним забезпеченням;


інтеграція з інструментами адміністрування Windows.

2.2. Апаратні обмеження при встановленні 32- та 64-бітних ОС
32-бітна ОС:


може бути встановлена практично на будь-якому процесорі;


обмеження по оперативній пам’яті — до ~4 ГБ;


не потребує апаратної підтримки віртуалізації.


64-бітна ОС:


потребує 64-бітного процесора;


обов’язкова підтримка апаратної віртуалізації (Intel VT-x або AMD-V);


підтримка цієї технології має бути увімкнена в BIOS/UEFI;


дозволяє використовувати більше 4 ГБ оперативної пам’яті.

1. Порівняння гіпервізорів типу 1 та типу 2
Гіпервізор типу 1 (bare-metal) працює безпосередньо на апаратному забезпеченні комп’ютера без використання хостової операційної системи. Він має вищу продуктивність, кращу стабільність і рівень безпеки, тому застосовується переважно у серверних, корпоративних та хмарних середовищах.
Гіпервізор типу 2 (hosted) працює поверх уже встановленої операційної системи як прикладна програма. Він простіший у встановленні та використанні, але залежить від хост-ОС, що знижує продуктивність. Використовується для навчання, тестування та роботи на персональних комп’ютерах.
2. Поняття «GNU GPL» та його основна концепція
GNU GPL (General Public License) — це ліцензія вільного програмного забезпечення, створена в межах проєкту GNU. Її основна концепція полягає у забезпеченні чотирьох свобод користувача: запускати програму, вивчати її код, змінювати програму та поширювати як оригінальні, так і змінені версії.
Ключовою особливістю GPL є принцип copyleft, який зобов’язує поширювати похідні продукти під тією ж ліцензією.
3. Суть програмного забезпечення з відкритим кодом
Програмне забезпечення з відкритим кодом — це ПЗ, вихідний код якого є відкритим та доступним для перегляду, вивчення, модифікації та поширення. Такий підхід забезпечує прозорість, підвищену безпеку, швидкий розвиток програм та незалежність від конкретного виробника.
4. Поняття дистрибутиву
Дистрибутив Linux — це комплект програмного забезпечення, який включає ядро Linux, системні утиліти, менеджер пакетів, бібліотеки та прикладні програми, об’єднані для зручного встановлення і використання операційної системи.
5. Задачі системного адміністрування в ОС Linux
На базі ОС Linux можна реалізувати такі задачі системного адміністрування:
керування користувачами та правами доступу;


адміністрування файлових систем;


налаштування мережі та мережевих сервісів;


керування процесами та ресурсами системи;


автоматизація задач за допомогою скриптів;


забезпечення безпеки та резервного копіювання;


розгортання серверних сервісів (web, FTP, mail).


6. Зв’язок між ОС Android та Linux
Операційна система Android базується на ядрі Linux, яке відповідає за керування процесами, пам’яттю, файловою системою та апаратними ресурсами. При цьому Android використовує власну програмну платформу та інтерфейс, що відрізняє його від класичних дистрибутивів Linux.
7. Основні можливості та сфера використання Embedded Linux
Embedded Linux — це спеціалізована версія Linux, призначена для вбудованих систем.
Основні можливості:
малий розмір системи;


висока стабільність;


підтримка різного апаратного забезпечення;


можливість глибокої кастомізації.


Сфера використання: маршрутизатори, смарт-телевізори, автомобільні системи, промислові контролери, IoT-пристрої.
8. Зміна типу завантаження Linux та відмінності CLI і GUI
Зміна режиму завантаження Linux здійснюється шляхом зміни рівня запуску системи.
Для переходу в текстовий режим (CLI, рівень 3):
sudo systemctl set-default multi-user.target

Для переходу в графічний режим (GUI, рівень 5):
sudo systemctl set-default graphical.target

CLI (Command Line Interface) — текстовий інтерфейс, який забезпечує точне керування системою та високу продуктивність.
GUI (Graphical User Interface) — графічний інтерфейс, зручний для користувачів-початківців та повсякденної роботи.
